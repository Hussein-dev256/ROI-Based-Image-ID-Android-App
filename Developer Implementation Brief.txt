## 1. Context & Purpose (Read This First)

I am building **ObjectID**, an AI-powered object recognition system.
The **Android app is the primary product**, and it uses **Imagga API** to identify objects from **user-selected regions of interest (ROI)** in images.

The purpose of this **web-based demo** is **not** to replace the Android app, but to:

* Reproduce the **exact Android workflow**
* Allow **recruiters and portfolio visitors** to experience the system
* Demonstrate **engineering thinking, UX maturity, and AI integration**
* Work without installing anything

## 4. My Current Pain Points (These MUST be fixed)

### 4.1 UI Quality

* Current UI feels **empty and unfinished**
* Too minimal ‚Üí looks like a prototype
* No clear hierarchy or guidance

üëâ The UI must have **visual density**, structure, and flow.

---

### 4.2 Image Handling (Major Issue)

* Uploaded images **expand uncontrollably**
* Makes annotation extremely difficult
* No fixed viewing stage

üëâ **Fix:**
Images must live inside a **bounded image stage / viewport**:

* Fixed max width & height
* Image scales to fit (maintain aspect ratio)
* Support zoom & pan **inside** the stage
* Never overflow the container

---

### 4.3 Annotation Canvas (Critical)

The annotation rectangle is currently:

* Not movable freely
* Not resizable
* Not interactive
* Not intuitive

üëâ **This is unacceptable for an ROI-based system**

The annotation canvas **must** support:

* Dragging (horizontal, vertical, diagonal)
* Resizing from all corners and edges
* Minimum & maximum size constraints
* Cursor feedback (move / resize)
* Semi-transparent fill
* Clear handles
* Smooth pointer-based interaction (mouse + touch)

This should feel like:

* Figma
* Photoshop
* Mobile crop tools
